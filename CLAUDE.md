# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Start local Ganache blockchain (required before migrate/test)
npm run devchain

# Compile smart contracts → build/contracts/
npm run compile

# Deploy contracts to local chain (use --reset to redeploy)
npm run migrate
npx truffle migrate --reset

# Run contract tests
npm test

# Run a single test file
npx truffle test test/main.test.js

# Serve frontend at localhost (after migrate completes)
npm run dev
```

## Architecture

**Stack:** Truffle + Ganache + Web3.js + plain HTML/CSS/JS frontend with MetaMask.

**Smart Contracts (`contracts/`):**
- `Main.sol` — active contract; stores a single `uint256` with `setValue`/`getValue`. Emits `ValueChanged` event.
- `refundContract.sol` — secondary exercise; owner-gated Ether transfers with a `receive()` payable function.

Only `Main` is deployed via `migrations/1_deploy_contracts.js` and tested.

**Frontend (`src/`):**
- Single-page vanilla JS app with no build tool.
- `js/app.js` loads the compiled ABI from `build/contracts/Main.json` at runtime — this file is generated by `compile`/`migrate` and must exist before the frontend works.
- MetaMask (`window.ethereum`) is the only auth mechanism; no backend.
- State lives in an IIFE-scoped object (`web3`, `account`, `contract`).

**Development network:** Ganache on `127.0.0.1:8545`. MetaMask must be pointed to this network.

**Workflow order:** `devchain` → `compile` → `migrate` → `dev` → connect MetaMask to localhost:8545.
